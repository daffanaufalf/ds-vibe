# Button

Buttons allow users to take actions, and make choices, with a single tap. They are essential building blocks for user interaction.

## When to Use

- **Solid**: Used for the primary action on a page. There should typically only be one primary button per screen.
- **Outline**: Used for secondary actions that are important but not the main focus.
- **Ghost**: Used for tertiary actions, like "Cancel" or for actions in toolbars/menus.
- **Link**: Used for actions that navigate or behave like links.
- **Destructive**: Used for irreversible actions like deleting or removing data.

## Accessibility

- The button uses proper `aria-disabled` and `aria-busy` attributes.
- Keyboard support: `Enter` and `Space` work as expected (native button behavior).
- Contrast ratios are checked against WCAG 2.1 AA standards.
- Font sizes and touch targets (minimum 44x44px for md/lg sizes) are optimized.
